Kernel 5 — 2D Blocktiling SGEMM

BM=64, BN=64, BK=8, TM=8, TN=8 · 64 threads/block · C = αAB + βC

§1 · Tile Hierarchy
A (in SMEM)
As[idx] = A[idx/BK·K + idx%BK]
B (in SMEM)
Bs[idx] = B[idx/BN·N + idx%BN]
C block tile (in registers)
BM·BN/(TM·TN) = 64·64/(8·8) = 64 threads
§2 · Thread Index Decomposition
threadIdx → (threadRow, threadCol)
threadRow = threadIdx.x / (BN/TN) // = threadIdx.x / 8
threadCol = threadIdx.x % (BN/TN) // = threadIdx.x % 8
64 threads laid out as 8 rows × 8 cols
each owning a TM×TN = 8×8 register tile
tiling the full BM×BN = 64×64 C block
C tile addressing (writeback)
row_C = TM · threadRow + i // i ∈ [0, TM)
col_C = TN · threadCol + j // j ∈ [0, TN)
C[…] = α·res[i·TN+j] + β·C[…]
C pointer is pre-offset to block tile:
C += blockIdx.x·BM·N + blockIdx.y·BN

Final index within block:
C[(TM·threadRow + i)·N + (TN·threadCol + j)]
Pointer advancement per outer loop step
A += BK → slide right BK cols along block row
B += BK·N → slide down BK rows along block col

Key identity exploited
A @ B = Σₖ col(A,k) ⊗ row(B,k)
SMEM → register loads (inner loop)
regA[i] = As[(threadRow·TM + i)·BK + dotIdx]
regB[j] = Bs[dotIdx·BN + threadCol·TN + j]
res[i·TN+j] += regA[i] · regB[j]
Why registers matter
Without regA/regB caches:
TM·TN = 64 SMEM reads per dotIdx step

With register caching:
TM + TN = 16 SMEM reads per dotIdx step
4× reduction in SMEM bandwidth
§4 · Loop Structure Summary
// outer loop: slide BKxBK tile along K dimension — K/BK = 512 iterations
for blckIdx in [0, K, step=BK):
  // all threads cooperatively fill SMEM (BMxBK As, BKxBN Bs)
  load As[BM×BK], Bs[BK×BN] from global → SMEM
  __syncthreads()

  // inner loop: BK outer-product steps
  for dotIdx in [0, BK):
    load regA[TM] ← col(As, dotIdx)
    load regB[TN] ← row(Bs, dotIdx)
    res[TM×TN] += regA ⊗ regB

  __syncthreads()
  A += BK   // next tile col
  B += BK·N  // next tile row
